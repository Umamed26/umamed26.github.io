我的跨平台写作方案：Obsidian + 坚果云

怎么做到随时随地创作。

发布 2026/3/5
分类 技术
系列 workflow
#tools

1. 写作的痛点：灵感总在不该来的时候来

作为一名半桶水的创作者，我偶尔最大的困扰从来不是没灵感，而是灵感来了，但笔记不在身边。

  • 坐在电脑前想写的时候，脑子一片空白。
  • 好不容易躺平在床上，或者在地铁里被挤成照片时，脑子里突然冒出绝妙的对话或情节转折——然后掏出手机打开某款App发现和电脑上的文稿完全割裂，的灵感已经在解锁屏幕的瞬间蒸发了一半。
  • 用U盘来回拷贝？石器时代的操作。同步盘冲突文件比我的草稿还多。而在Windows上跑iCloud就像去南极随便抓一只凑企鹅学飞行，勉强扑腾两下就摔了。

我需要的，是一个打开就能写，写完不用管，换设备继续写的无缝环境。——一个能包容我懒癌晚期、拖延症末期以及间歇性码字冲动的智能保姆。

于是，我搭建了一套基于Obsidian + 坚果云的跨平台写作系统。它不一定最酷，但足够稳定、足够自由，而且完全免费（如果你利用好Obsidian的同步插件生态并且愿意花5分钟动动手指的话）。

2. 为什么是这个组合？

在选择工具时，我对比过几种方案：

  • Obsidian官方同步服务： 体验很丝滑，但需要付费订阅，对学生党/月卡党不太友好。（每月几美元省下来够我买几箱可乐了。）
  • iCloud： 苹果用户的心头好，但在Windows上表现得像个蹒跚学步的婴儿，且容易因同步机制导致笔记冲突或丢失。何况我也没那套全家桶。
  • Git： 适合那些即使写个言情小说也要搞Branch和Pull Request的硬核人士。毕竟我只是想写东西，不想思考如何解决合并冲突。。

最终我选择了 Obsidian（负责写）+ 坚果云（负责传）+ Remotely Save 插件（负责连接） 的组合。

为什么是坚果云？

  • 速度快： 国内服务器，上传下载基本跑满带宽。
  • 稳定： 很少遇到同步冲突。
  • 便宜量大： 免费版的带宽对个人来讲够用了。
  • WebDAV协议： 这是关键。它像一个通用语言，让Obsidian插件能和坚果云顺畅对话。不需要学什么复杂的API，也不用看那些把文档写得像天书一样的开发指南。

3. 手把手配置指南

如果你也想搭建这套系统，跟着下面几步走就好。

第一步：安装核心插件——Remotely Save

在Obsidian的社区插件市场里，搜索并安装 Remotely Save。 （如果你无法访问插件市场，可以从 GitHub 手动下载安装。）

第二步：获取坚果云的“第三方应用密码”

这是最容易被卡住的一步，但其实很简单。

  1. 登录坚果云网页版，进入【账户信息】。
  2. 点击【安全选项】，找到【第三方应用管理】。
  3. 点击【添加应用】，随便起个名字（比如“怎么防止自己挖一万个坑不填”）。
  4. 系统会生成一串服务器地址、账号、应用密码。注意：这里生成的是应用密码，不是你的登录密码。把它复制下来，保存好。丢了也没事，无非是再生成一次。

第三步：配置 Remotely Save 插件

  1. 回到 Obsidian，打开 Remotely Save 的设置。
  2. 远程服务类型： 选择 “WebDAV”。
  3. WebDAV 链接： 粘贴刚才复制的服务器地址。
  4. 用户名： 粘贴刚才复制的账号（通常是你的坚果云邮箱）。
  5. 密码： 粘贴刚才复制的应用密码（不是你的邮箱密码，真的。）。
  6. 点击【检查】，如果提示“成功”，说明配置好了。
  7. 在【同步设置】里，建议开启 “定时自动同步”（比如每5分钟一次），并打开 “启动时同步” 和 “关闭时同步”。这样基本可以做到无感同步。

第四步：在手机上配置

在手机端（iOS或Android）安装Obsidian，新建一个空库，然后安装Remotely Save插件，填入完全一样的信息。稍等片刻，你的小说库就会从云端下载到手机里。

4. 一些实用的小建议

使用这套方案几个月后，我总结了几点经验，也许对你有帮助：

  1. 关于同步频率： 不建议设置“实时同步”（如果插件支持的话），每1-5分钟同步一次足够了。写作时疯狂同步反而会干扰思路，甚至可能造成性能问题或者烧流量。除非你喜欢看着右下角的通知不停冒，并以此为乐。
  2. 关于冲突： 虽然坚果云+Remotely Save很稳定，但偶尔（比如在信号不好的地方）可能会出现同步冲突。Obsidian会自动生成冲突文件（如“文件名 的冲突副本”）。我一般每周清理一次库，检查有没有这类文件，确认无误后删除。
  3. 关于备份： 坚果云本身有文件历史版本功能，可以找回30天内修改过的版本。这相当于多了一层保险。如果写得不满意想回退，去坚果云网页版找回旧版本就行。（这个功能救过我很多次，让我有勇气把第一章删了重写二十遍。是的，我还在写第一章。）
  4. 关于手机写作体验： 手机端只适合记录“那个人死了”这种突发灵感。至于他怎么死的，死状如何，周围人的心理描写，请滚回电脑前写。毕竟很显然，手机能做的，电脑能做的更好，而手机做得最出色的是让你在想写作时点开短视频。

5. 结语

这套Obsidian + 坚果云的方案，没有华丽的界面，也没有昂贵的订阅费，它只是安静地待在那里，让我的小说库始终处于最新状态，让我在任何想写的时候都能立刻开始。

如果你也是 Obsidian 用户，并且正在寻找一套跨平台同步方案，不妨试试这个组合。如果你有更好的写作工具搭配，欢迎在评论区交流。

如果你在配置过程中遇到了问题，也可以留言，我会在能力范围内帮你解答——不过大概率我只会回一句“重启试试”。
